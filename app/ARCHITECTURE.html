Executive Summary

This document defines the complete technical architecture for the Steadfast Student Services web platform — a full-stack application designed to automate the student enrollment journey from first inquiry through visa approval. The platform centers on three core pillars: intelligent automation (reducing manual advisor work by 70%), a 24/7 AI chatbot powered by Google Gemini 1.5 Flash (free tier), and a student-first UX engineered for repeat engagement and trust.

Upgrade Path

The chatbot module is built as an abstracted service layer. Switching from Gemini to Claude (or any other model) requires changing one environment variable and one API adapter file — no frontend changes needed. This means zero lock-in and a clean upgrade path when revenue supports it.

Section 9

AI Chatbot — Behavior Design

The chatbot is a floating widget on every page. It is not just a FAQ bot — it is a guided enrollment advisor that qualifies leads, collects contact info, and books consultations automatically.

System Prompt Strategy

Gemini receives a carefully crafted system prompt that establishes the bot as "Sofia" — a Steadfast Student Services enrollment advisor. The prompt includes:

Company overview and services (never reveal all pricing upfront)
Risk tier definitions for each country region
Common visa Q&A (F-1, M-1, J-1 processes)
Instructions to collect email before giving detailed advice
Instructions to recommend booking a consultation for complex cases
Tone: warm, professional, encouraging — never overwhelming
Escalation: "Let me connect you with a human advisor" for edge cases

Engagement Triggers (Zero Cost, High Conversion)

Time on page > 45s: Chatbot opens with "Hi! Curious about studying in the U.S.?"
Pricing page visit: "I can help you find the right package — what's your home country?"
3rd page view: "Looks like you're doing your research — want a quick personalized assessment?"
Return visitor: "Welcome back! Ready to take the next step?"
Exit intent: "Before you go — get your free visa eligibility score (2 minutes)"
Chat > 5 messages: "Would you like to book a free 30-minute call with an advisor?"

Section 12

Key Design Decisions

Why Next.js over plain HTML?

The existing documents are standalone HTML files — perfect for printable reports. But a dynamic web app requires real-time data, authentication, file uploads, and API calls. Next.js provides all of this with a single framework, Vercel deploys it for free, and the developer experience is excellent for future contractors or team members.

Why Supabase over Firebase?

Supabase is PostgreSQL — a real relational database with joins, triggers, and row-level security built in. It also includes auth, file storage, real-time subscriptions, and Edge Functions. The free tier is generous. PostgreSQL ensures your student data is structured, queryable, and easily exportable — critical for compliance and reporting.

Why Gemini over Claude for chatbot?

Gemini 1.5 Flash has a completely free tier (1M tokens/day, 15 RPM). For a startup with unpredictable early traffic, this means zero chatbot costs for months. Claude Haiku 4.5 has no free tier — even at $1/MTok, it costs money from day one. The architecture abstracts the AI provider so switching to Claude when revenue grows is a one-line change.

Why Cal.com over Calendly?

Cal.com is open-source with a generous free hosted tier that includes unlimited event types, Zoom/Google Meet integration, and webhook support for automation triggers. Calendly's free tier is limited to one event type. Cal.com's webhook fires on every booking, feeding directly into the Supabase automation engine.
